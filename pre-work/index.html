    Preparación¶

    Los laboratorios de la workshop son notebooks de Jupyter. Los notebooks pueden ejecutarse:

    • Localmente en tu equipo (para el cual necesitarás instalar algunas dependencias) O
    • De forma remota en Google Colab

    Sigue las instrucciones en una de las siguientes secciones según cómo prefieras ejecutarlos.

    Ejecución de notebooks localmente¶

    Si quieres ejecutar los notebooks de los laboratorios localmente en tu ordenador, se recomienda contar con:

    • Un ordenador o portátil: mínimo 8 GB de RAM (16 GB recomendado)
    • Conocimientos de Git y Python: Lo suficiente para instalar dependencias y ejecutar Jupyter Notebooks
    • (Opcional) Un IDE como VS Code o PyCharm para editar y ejecutar los notebooks comodamente.

    Si alguna de estas no es el caso, te recomendamos ir a la sección Ejecución de notebooks de forma remota (Colab).

    Para ejecutar los notebooks de los laboratorios localmente en tu equipo, sigue estos pasos:

    • Prerrequisitos locales
    • Clonar el repositorio de la workshop
    • Instalar Jupyter

    Prerrequisitos locales¶

    • Git
    • Python 3.10, 3.11 o 3.12

    Clonar el repositorio de la workshop de Docling¶

    Clona el repositorio de la workshop y entra en el directorio del repositorio.

    git clone https://github.com/simonsanvil/docling-workshop-pycones-2025
cd docling-workshop-pycones-2025

    Instalar Jupyter¶

    Usa un entorno virtual

    Antes de instalar dependencias y para evitar conflictos en tu entorno, es aconsejable utilizar un entorno virtual (venv).

    1. Crea un entorno virtual:

      python3 -m venv --upgrade-deps --clear venv

    2. Activa el entorno virtual ejecutando:

      source venv/bin/activate

    3. Instala Jupyter Notebook en el entorno virtual:

      python3 -m pip install --require-virtualenv notebook ipywidgets

      Para más información, consulta las instrucciones de instalación de Jupyter.

    4. Para abrir un notebook en Jupyter (con el entorno virtual activo), ejecuta:

      jupyter notebook <notebook-file-path>

      Alternativamente, si tienes instalado VS Code, puedes habilitar la extensión de Jupyter Notebooks para VS Code (instrucciones aquí) y abrir los notebooks directamente en VS Code.

    Uso de los modelos de Granite con Replicate¶

    En el laboratorio 3 utilizaras modelos de IA generativa Granite para construir un sistema RAG multimodal. Para ello usaremos Replicate, una plataforma en la nube que aloja y sirve modelos de IA por ti.

    Instrucciones para configurar Replicate¶

    1. Crea una cuenta en Replicate. Necesitarás una cuenta de GitHub para hacerlo.

    2. Añade saldo gratis a tu cuenta de Replicate: Para facilitar probar los modelos en Replicate, utiliza este enlace para añadir una pequeña cantidad de crédito a tu cuenta cortesía de la comunidad de IBM Granite .

    3. Crea un token de API de Replicate.

    4. Establece tu token de API de Replicate como una variable de entorno en la terminal donde vayas a ejecutar el notebook:

      export REPLICATE_API_TOKEN=<your_replicate_api_token>

    Ejecución de notebooks de forma remota (Colab)¶

    Para ejecutar los notebooks de los laboratorios de forma remota usando Google Colab, sigue estos pasos:

    • Prerrequisitos de Colab
    • Servir los modelos de Granite con Replicate

    Velocidad de ejecución de notebooks en Colab

    En Colab, el runtime por defecto usa CPUs. Considera cambiar el tipo de entorno en Colab para aumentar la velocidad de ejecución, especialmente si tienes otras limitaciones como una conexión lenta. En la barra de navegación, selecciona Runtime->Change runtime type y elige aceleración por GPU o TPU.

    Prerrequisitos de Colab¶

    • Google Colab requiere una cuenta de Google con la que hayas iniciado sesión.

    Servir los modelos de Granite con Replicate¶

    En el laboratorio 3 utilizaras modelos de IA generativa Granite para construir un sistema RAG multimodal. Para ello usaremos Replicate, una plataforma en la nube que aloja y sirve modelos de IA por ti.

    Instrucciones para configurar Replicate¶

    1. Crea una cuenta en Replicate (necesitarás una cuenta de GitHub).

    2. Añade saldo gratis a tu cuenta de Replicate: Para facilitar probar los modelos Granite en Replicate, utiliza este enlace para añadir una pequeña cantidad de crédito a tu cuenta cortesía de la comunidad de IBM Granite .

    3. Crea un API token de Replicate.

    4. Añade tu token de API de Replicate al gestor de secretos de Colab para guardarlo de forma segura. Abre Google Colab y haz clic en la pestaña 🔑 Secrets del panel izquierdo. Pulsa "New Secret" e introduce REPLICATE_API_TOKEN como clave; pega tu token en el campo de valor. Activa el botón de la izquierda para permitir que el notebook acceda al secreto.
